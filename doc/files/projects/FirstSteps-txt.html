Based on Applications

The easiest way to write a new application is to copy one of the examples and adapt it to the application’s needs.  Good candidates for starting the development are the stream examples (see Stream Example (PIC32) and Stream Examples (Windows/Linux)).

Based on Templates

An alternative to the applications are the templates.  Unlike the examples the templates also contain commented alternatives.

The page Templates contains more details on how to use the templates.

Start from scratch

It is also possible to start a project from scratch or to add GestIC-device support directly to an existing application.

Windows applications can directly use the installed binaries.  Depending on whether the application should link dynamically or statically against the GestIC API one has to link the applications against api/lib/gestic.lib or api/lib-static/gestic.lib.

The installer of the GestIC SDK adds the api directory as the GESTIC_SDK_API_PATH environment variable (After installation of the GestIC SDK one restart is required before applications can use it).  By adding %GESTIC_SDK_API_PATH%/lib or %GESTIC_SDK_API_PATH%/lib-static (in case of static linking) as a library directory one could simply add gestic.lib as a dependency to the linker.

The include path for the API headers could be added as %GESTIC_SDK_API_PATH%/include to the compiler configuration.

After this setup the API could be used in source files.

If the new project is a Windows or Linux project the setup is now complete and the new application could be developed as described in gestic_t Struct, Data Retrieval and Real Time Control and Firmware Version and Update.

Projects for PIC32 or other embedded devices require additional platform specific code like the implementation of the input/output functionality.  The next page Customizing the API describes how this could be implemented.
