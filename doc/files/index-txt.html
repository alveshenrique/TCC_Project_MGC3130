Introduction

The GestIC SDK is a software development kit package supporting full interface and interaction between GestIC devices and your application.  It provides a high level API for the communication and the use of the features of the device.  It also contains complete documentation and applications examples which demonstrate how to use the API in different contexts and on different hardware platforms.  The API is fully tested and ready to use.  It saves your development time and gives you an abstraction layer of the GestIC I2C communication protocol low level.  It also provides ready functions to decode and configure all MGC3130 data outputs.  The API is intended to be used as it is.  Customer can build their own applications based on the provided pre-configured projects.

This version of the SDK contains binaries for Windows and the source code that builds for Linux and the PIC32MX without modification.  Customizing the API section provides instructions on how to use the API on different platforms and with different configurations.

The page Compilation has instructions on how to compile the SDK on the different platforms API Usage, First Steps, Data Retrieval and Real Time Control and Firmware Version and Update contain instructions on how to use the prebuilt binaries inside an application.  Finally Customizing the API describes how to port and adjust the API for use in embedded devices.

Feature Overview

The GestIC SDK consists of a single Application Programming Interface (API), including:

  • Core functions of the API
  • Firmware Update functions
  • Firmware Version query functions
  • Real Time Control functions
  • Data Retrieval functions

Supported Platforms

With the GestIC SDK, you can build apps that target PC-based or embedded applications.

  • Windows
  • Linux
  • PIC32

Getting started

The GestIC SDK is provided as an exe file.  After downloading the SDK, you will need to install on your Windows PC.  Then, you copy the installed directory to any other platform like Linux.

Discovering the SDK

The SDK is composed by four folders

apiThis folder contains the source code for the API.
appsThis folder contains the source code and pre-configured projects for the examples.  To view a list of all SDK examples; check the List of Apps page.
docSDK documentation
utilitiesadditional useful tools.

Setting up Projects

The package provides pre-configured projects for all supported platforms.  Check the Compilation page for more information.

How to use SDK

The API is ready library providing full support of the GestIC devices features.  The user has to select which platform he wants to use and then refer to the corresponding provided examples.  The examples can be tailored and modified to the main application requirements.

There are different ways to start with the development of a new GestIC Application.
The primary taks of the MGC3130 is detection of Position and/or Gestures.
The Console Application retrieves more data than the Stream Examples (Windows/Linux) and allows to trigger a few Real Time Controls.
The programmer application demonstrates how to update the Loader Library and the GestIC Library.
The dynamic stream example demonstrates how to use the Dynamic API to use the API when it is provided as an DLL.
The static stream example demonstrates how the Static API could be used if the application is built together with the API or if it is linked statically against the API.
The programmer application is a self-contained firmware updater for the PIC32 USB Starter Kit II (http://www.microchip.com/DevelopmentTools/dm320003-2) and the I/O Expansion Board (http://www.microchip.com/DevelopmentTools/dm320002).
The programmer application is a self-contained firmware updater for the PIC32 USB Starter Kit II (http://www.microchip.com/DevelopmentTools/dm320003-2) and the I/O Expansion Board (http://www.microchip.com/DevelopmentTools/dm320002).
The MGC Image Conversion tool (MGCConv) allows extraction of firmware-images as C-structures from ENZ-Files.
The CDC-Driver is required for communication with Hillstar or any other customer application using Microchips Hillstar I²C to USB-Bridge.
