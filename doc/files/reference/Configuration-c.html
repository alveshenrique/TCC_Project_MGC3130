Macros

GESTIC_CUSTOM

Triggers the inclusion of gestic_custom.h into the API.

This macro has to be provided via the command line arguments of the compiler.  If defined gestic_api.h includes gestic_custom.h which has to be at one of the include paths.

gestic_custom.h can override the default configuration and provide an application specific custom API.  Customizing the API describes how this can be done.

Important

Both the API and the application have to be compiled with the same gestic_custom.h in place.  Otherwise it will show undefined behavior.

GESTIC_IO

GESTIC_IO allows to set or override the IO-implementation.

GESTIC_IO_CUSTOMThe application provides its own implementation of the IO
GESTIC_IO_CDC_SERIALDefault Implementation on Windows and Linux.  Connects to the MGC3130 via the IÂ²C to USB Bridge.  (not available on embedded platforms)

See Custom IO Implementation for a description of how to provide a custom IO-implementation.

When GESTIC_IO is set to GESTIC_IO_CUSTOM make sure to define the correct synchronization mode for message handling if gestic_message_handle is called outside of gestic_message_receive (the synchronization modes are GESTIC_SYNC_INTERRUPT and GESTIC_SYNC_THREADING).

GESTIC_API_DYNAMIC

Has to be set when compiling the API as a DLL or when using it as DLL.  This macro has to be provided to the compiler as an command line argument.

GESTIC_HAS_DYNAMIC

If this macro was defined in the compilers command line or inside gestic_custom.h the Dynamic API will be available even when using the Static API.

GESTIC_API_EXPORT

This macro has to be only used when compiling the API as a DLL.  When using the API as a DLL or linking statically against this API this macro should be absent.  It has to be provided to the compiler as an command line argument.

Macros

GESTIC_SYNC_INTERRUPT

This macro specifies that gestic_message_handle is called during an interruption of the normal operation.  This is typically done from within a low priority ISR to avoid message-loss while performing time consuming operations.

See also

Message handling modes, GESTIC_SYNC_THREADING

GESTIC_SYNC_THREADING

This macro specifies that gestic_message_handle is called from within a different thread than the normal API functions.  Here a bidirectional synchronization is needed to avoid race conditions.

Threading IO mode allows to do IO completely asynchronosly from the rest of the application.

See also

Message handling modes, GESTIC_SYNC_INTERRUPT

Macros

GESTIC_MALLOC

Defaults to the function malloc from stdlib.h.

It is only required for the Dynamic API.

GESTIC_FREE

Default to the function free from stdlib.h.

It is only required fro the Dynamic API.

GESTIC_MEMSET

Defaults to the function memset from string.h.

GESTIC_MEMCPY

Defaults to the funczion memcpy from string.h.

GESTIC_SYNC_INIT

Intializes the mutex or another appropriate synchronization object.

Only needed when using Asynchronous message handling.

GESTIC_SYNC_RELEASE

Releases the mutex or another appropriate synchronization object.

Only needed when using Asynchronous message handling.

GESTIC_SYNC_LOCK

Locks the synchronization object.

Only needed when using Asynchronous message handling.

GESTIC_SYNC_UNLOCK

Unlocks the synchronization object.

Only needed when using Asynchronous message handling.

GESTIC_BAD_DATA

Is called when incoming data doesn’t match to the protocol.

FUNCThe function which detected the bad data.
MSGShort description of what was expected
VALUEThe value that didn’t fit to the expectation

The default implementation of this macro does nothing.  It might be usefull to provide an implementation for debugging broken communication.

GESTIC_ASSERT

Defaults to assert from assert.h

Only used to ensure correct use of the API.  It checks whether the correct and initialized arguments are given and if functions that communicate with an device are only called between an succesfull call to gestic_open and gestic_close.

GESTIC_SLEEP

Only needed for impelmentation of communciation with the CDC Serial bridge.
